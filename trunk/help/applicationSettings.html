Application Settings Dialog

The application settings dialog allows the user to configure the behaviour of the application. The settings have the following meaning:

  • Time between displaying two RSS messages: This value specifies the base time in minutes that elapses between two tooltip pop-ups with new RSS messages. The base time only applies between two messages as long as messages are available in the channel with the highest effective priority. As this channel has no more messages to display, the timespan between 2 RSS messages will increase corresponding to the decrease in effective priority.
  • Intervall for retrieving new RSS messages: This value specifies the time in minutes between two retrievals of the RSS messages via the network. Thus, after each such intervall, all RSS channels are retrieved.
  • Timespan to display balloon pop-up: This value specifies the time intervall in seconds, which a new RSS messages is displayed within the balloon tooltip. Consider that the Windows operating system controls the pop-up and may change the timing. I.e. the minimum timespan is set to 10 seconds by the OS and the display intervall will be extended if there is no user activity (e.g. using the mouse or keyboard).
  • Start in pause mode: As this is described in the usage section BalloonRSS can be put into pause mode. If this checkbox is selected BalloonRSS will start in pause mode. You need to click the program item in the task bar to activate it then.
  • Use channel name as pop-up title: Each displayed RSS message has two sections: the title and the content. Normally the title of the RSS message is displayed in the title section and its content is displayed below. If this checkbox is selected BalloonRSS will display the title of the RSS channel (instead of the RSS message) in the title field and the RSS message content below. BalloonRSS will not display the RSS message title then.
  • Number of entries in message history: This value specifies the number of lines in the RSS message history dialog.
  • Priority influence of message clicks: BalloonRSS calculates the effective priority of the RSS channels based on 2 inputs: the pre-defined priority of the channel and the number of user clicks onto RSS messages of this channel. This slider control enables the user to specify the influence of those 2 input characteristics. If the slider is put on the left side the influence of the pre-defined priority value is increased. If ths slider is put to the right the click rate has higher influence. You can immediately check the results in the channel information dialog.
  • Check for application updates: If this checkbox is selected BalloonRSS will check for potential updates on the SourceForge server. If a new version is available this will be reported as an message popup.
  • Report network errors: If this checkbox is selected BalloonRSS will report any network error during retrieval of the RSS channels as an message popup. If the checkbox is not selected network errors will be ignored without notifying the user.

All settings are kept on a per-user basis. Therefore a change in the settings will only change the program behaviour of the current user.
